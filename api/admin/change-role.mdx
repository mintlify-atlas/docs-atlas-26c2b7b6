---
title: "Change User Role"
api: "PATCH /api/admin/users/:id/role"
description: "Toggle a user's admin status between admin and regular user"
---

## Authentication

This endpoint requires:
- Valid JWT token (via cookie)
- Admin role (`isAdmin: true`)

## Path Parameters

<ParamField path="id" type="string" required>
  The MongoDB ObjectId of the user whose role should be changed
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success message indicating the new role status
</ResponseField>

<ResponseField name="listing" type="object">
  The updated user object (Note: field name is 'listing' in current implementation)
  
  <ResponseField name="_id" type="string">
    User ID
  </ResponseField>
  
  <ResponseField name="username" type="string">
    Username
  </ResponseField>
  
  <ResponseField name="email" type="string">
    User email
  </ResponseField>
  
  <ResponseField name="phone" type="string">
    Phone number
  </ResponseField>
  
  <ResponseField name="isAdmin" type="boolean">
    Updated admin status
  </ResponseField>
  
  <ResponseField name="avatar" type="object">
    Avatar information
  </ResponseField>
  
  <ResponseField name="createdAt" type="string">
    Account creation timestamp
  </ResponseField>
</ResponseField>

## Implementation Details

- Toggles the user's `isAdmin` field using `!user.isAdmin`
- No request body required - automatically toggles current status
- Uses `user.save()` to persist the change
- Returns a dynamic message based on the new admin status
- Admins can promote regular users to admin or demote admins to regular users

<Warning>
  Be cautious when changing user roles. Ensure you don't accidentally demote yourself or remove all admin users from the system.
</Warning>
