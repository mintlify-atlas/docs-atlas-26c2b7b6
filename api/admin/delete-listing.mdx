---
title: "Delete Listing"
api: "DELETE /api/admin/listings/:id"
description: "Permanently delete a listing from the database"
---

## Authentication

This endpoint requires:
- Valid JWT token (via cookie)
- Admin role (`isAdmin: true`)

## Path Parameters

<ParamField path="id" type="string" required>
  The MongoDB ObjectId of the listing to delete
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<RequestExample>
```bash cURL
curl -X DELETE "https://api.boundry.com/api/admin/listings/64a1b2c3d4e5f6789abcdef0" \
  -H "Cookie: jwt=YOUR_JWT_TOKEN"
```

```javascript JavaScript
const listingId = '64a1b2c3d4e5f6789abcdef0';

const response = await fetch(`/api/admin/listings/${listingId}`, {
  method: 'DELETE',
  credentials: 'include'
});

const data = await response.json();
```
</RequestExample>

<ResponseExample>
```json 200 Success
{
  "message": "Listing successfully deleted"
}
```

```json 404 Not Found
{
  "message": "Listing not found. It may have already been deleted."
}
```

```json 401 Unauthorized
{
  "message": "Token Authentication Failed :: Token not found"
}
```

```json 403 Forbidden
{
  "message": "Access Denied :: Admin permissions required"
}
```

```json 500 Server Error
{
  "message": "Listing deletion failed :: Error details"
}
```
</ResponseExample>

## Implementation Details

- Performs a **hard delete** using `findByIdAndDelete`
- Listing is permanently removed from the database
- Returns 404 if the listing doesn't exist or was already deleted
- This action cannot be undone
- Use this for listings that violate policies or contain inappropriate content
- For reversible moderation, use the reject endpoint instead

<Note>
  This is a permanent deletion. Consider implementing soft delete functionality for listings that may need to be recovered later.
</Note>
