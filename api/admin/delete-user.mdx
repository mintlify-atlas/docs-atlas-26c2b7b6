---
title: "Delete User"
api: "DELETE /api/admin/users/:id"
description: "Permanently delete a user from the system"
---

## Authentication

This endpoint requires:
- Valid JWT token (via cookie)
- Admin role (`isAdmin: true`)

## Path Parameters

<ParamField path="id" type="string" required>
  The MongoDB ObjectId of the user to delete
</ParamField>

## Response

<ResponseField name="message" type="string">
  Success or error message
</ResponseField>

<ResponseField name="user" type="object">
  The deleted user object (included on success)

    User ID
  </ResponseField>
  
  <ResponseField name="username" type="string">
    Username
  </ResponseField>

  <ResponseField name="phone" type="string">

## Implementation Details

- Performs a **hard delete** using `findByIdAndDelete`
- User is permanently removed from the database
- Returns the deleted user object on success
- Returns 404 if the user doesn't exist
- This action cannot be undone
- Associated listings may become orphaned (consider cascading deletes in production)

<Warning>
  This is a permanent deletion. Consider the following before deleting a user:
  - Their listings will remain in the database but may become orphaned
  - This action cannot be undone
  - Don't delete yourself while logged in as admin
  - Consider implementing soft delete for better data integrity
</Warning>

<Note>
  The current implementation does not cascade delete user listings. You may need to manually clean up or reassign listings created by deleted users.
</Note>
