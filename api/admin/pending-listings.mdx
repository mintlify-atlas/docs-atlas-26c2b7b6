---
title: "Get Pending Listings"
api: "GET /api/admin/listings/pending"
description: "Retrieve all listings awaiting admin approval with pagination support"
---

## Authentication

This endpoint requires:
- Valid JWT token (via cookie)
- Admin role (`isAdmin: true`)

All admin endpoints use two middleware layers:
1. `authenticateToken` - Verifies JWT token from cookies
2. `authorizeAdmin` - Checks user has admin privileges

## Query Parameters

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="number" default="10">
  Number of listings per page (max 100)
</ParamField>

## Implementation Details

- Listings are filtered by `isApproved: false`
- Results are sorted by `createdAt` in ascending order (oldest first)
- Creator information is populated with username, avatar, phone, and email
- Limit parameter is capped at 100 to prevent excessive data retrieval
- Returns 404 if no pending listings exist
