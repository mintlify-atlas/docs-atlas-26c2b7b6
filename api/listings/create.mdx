## Overview

Creates a new property listing. Requires authentication and supports uploading up to 10 images.

<Note>
  All newly created listings are set to `isApproved: false` and require admin approval before appearing in public listings.
</Note>

## Authentication

**Required:** Yes (JWT token in cookies)

The `createdBy` field is automatically set from the authenticated user.

## Content Type

```
multipart/form-data
```

## Examples

### Create a listing with form data

```bash
curl -X POST https://api.boundry.app/api/listings \
  -H "Cookie: jwt=YOUR_JWT_TOKEN" \
  -F "title=Luxury Villa in Dubai Marina" \
  -F "description=Stunning 4-bedroom villa with sea views" \
  -F "price=2500000" \
  -F "location[city]=dubai" \
  -F "location[area]=Dubai Marina" \
  -F "propertyType=villa" \
  -F "status=for-sale" \
  -F "bedrooms=4" \
  -F "bathrooms=3" \
  -F "areaSqft=3500" \
  -F "images=@/path/to/image1.jpg" \
  -F "images=@/path/to/image2.jpg" \
  -F "images=@/path/to/image3.jpg"
```

## Implementation Details

- Implemented in `controllers/listingController.js:122`
- Route defined in `routes/listingRoutes.js:18`
- Uses `upload.array("images", 10)` middleware for handling up to 10 images
- Images are stored temporarily in `./public/temp` before Cloudinary upload
- All images are uploaded to Cloudinary in parallel using `Promise.all`
- Failed image uploads are filtered out; at least one successful upload is required
- Location fields are automatically lowercased/trimmed per schema validation
