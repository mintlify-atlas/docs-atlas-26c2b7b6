api: "GET /api/listings/:id"

---

## Overview

Fetches detailed information about a specific listing. Only approved listings are accessible to the public.

## Path Parameters

<ParamField path="id" type="string" required>
  The MongoDB ObjectId of the listing
</ParamField>

## Authentication

**Required:** No

Public endpoint, but only returns approved listings.

  Unique listing identifier
</ResponseField>

<ResponseField name="title" type="string">

</ResponseField>

<ResponseField name="description" type="string">

## Error Responses

### Listing Not Found (404)

```json
{
  "message": "Listing not found"
}
```

### Listing Not Approved (403)

```json
{
  "message": "Listing not approved"
}
```

## Implementation Details

- Implemented in `controllers/listingController.js:105`
- Route defined in `routes/listingRoutes.js:28`
- The `createdBy` field is populated with full contact information (`username`, `avatar`, `phone`, `email`)
- Only listings with `isApproved: true` are returned to public users
- Unapproved listings return a 403 Forbidden error
