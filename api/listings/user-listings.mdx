## Overview

Retrieves all listings created by the currently authenticated user. Supports filtering by status and searching by title.

## Authentication

**Required:** Yes (JWT token in cookies)

The `createdBy` filter is automatically set from the authenticated user.

## Query Parameters

<ParamField query="search" type="string" default="">
  Case-insensitive search in listing titles
</ParamField>

<ParamField query="status" type="string" default="all">
  Filter by listing status
  
  **Allowed values:**
  - `all` - All listings
  - `active` - Approved listings that haven't been sold/rented
  - `pending` - Not yet approved by admin
  - `sold` - Sold or rented listings
</ParamField>

<ParamField query="page" type="number" default="1">
  Page number for pagination
</ParamField>

## Examples

### Get all user's listings

```bash
curl http://localhost:5000/api/listings/my-listings \
  -H "Cookie: jwt=YOUR_JWT_TOKEN"
```

### Get only pending listings

```bash
curl "http://localhost:5000/api/listings/my-listings?status=pending" \
  -H "Cookie: jwt=YOUR_JWT_TOKEN"
```

### Get active listings with search

```bash
curl "http://localhost:5000/api/listings/my-listings?status=active&search=villa" \
  -H "Cookie: jwt=YOUR_JWT_TOKEN"
```

### Get sold/rented listings

```bash
curl "http://localhost:5000/api/listings/my-listings?status=sold" \
  -H "Cookie: jwt=YOUR_JWT_TOKEN"
```

## Status Filter Logic

### `active`
Listings where:
- `isApproved: true`
- `acquiredStatus: null`

### `pending`
Listings where:
- `isApproved: false`
- `acquiredStatus: null`

### `sold`
Listings where:
- `acquiredStatus` is `"sold"` or `"rented"`

### `all`
All listings owned by the user, regardless of approval or acquisition status.

## Implementation Details

- Implemented in `controllers/listingController.js:43`
- Route defined in `routes/listingRoutes.js:19`
- Fixed limit of 5 listings per page
- Results sorted by `createdAt` in descending order (newest first)
- The `createdBy` field is not populated (returns user ID only)
- Search matches against listing title with case-insensitive regex
