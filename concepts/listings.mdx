Listings are property advertisements created by users. Each listing goes through an approval workflow before becoming publicly visible.

## Listing Schema

The Listing model defines property structure and validation:

```javascript
// From models/listingModel.js:3-94
const listingSchema = mongoose.Schema({
  title: { type: String, required: true, trim: true },
  description: { type: String, required: true, trim: true },
  price: { 
    type: Number, 
    required: true, 
    min: [1, "Price needs to be higher than 0"] 
  },
  location: {
    city: { type: String, required: true, trim: true, lowercase: true },
    area: { type: String, required: true, trim: true },
  },
  propertyType: {
    type: String,
    required: true,
    lowercase: true,
    enum: {
      values: ["house", "apartment", "villa", "studio", "office"],
      message: "{VALUE} is not a valid property type",
    },
  },
  status: {
    type: String,
    required: true,
    lowercase: true,
    enum: {
      values: ["for-sale", "for-rent"],
      message: "{VALUE} is not a valid status",
    },
    default: "for-sale",
  },
  acquiredStatus: {
    type: String,
    lowercase: true,
    enum: ["sold", "rented"],
  },
  bedrooms: { 
    type: Number, 
    required: true, 
    min: [1, "There must be atleast 1 bedroom"] 
  },
  bathrooms: { 
    type: Number, 
    required: true, 
    min: [1, "There must be atleast 1 bathroom"] 
  },
  areaSqft: { type: Number, required: true },
  images: [{
    url: { type: String, required: true },
    publicId: { type: String, required: true },
  }],
  createdBy: { 
    type: mongoose.Schema.Types.ObjectId, 
    ref: "User", 
    required: true 
  },
  isApproved: { type: Boolean, default: false },
  createdAt: { type: Date, default: Date.now },
});
```

## Listing Lifecycle

<Steps>
  <Step title="Creation">
    User creates a listing with property details and images
    
    - `isApproved` is set to `false` by default
    - Images uploaded to Cloudinary
    - `createdBy` set to authenticated user's ID
  </Step>
  
  <Step title="Pending Review">
    Listing is invisible to public, awaiting admin review
    
    - Only visible to creator and admins
    - Appears in admin pending listings queue
  </Step>
  
  <Step title="Admin Decision">
    Admin approves or rejects the listing
    
    **If approved:**
    - `isApproved` set to `true`
    - Listing becomes publicly visible
    
    **If rejected:**
    - `isApproved` remains `false`
    - Listing stays hidden from public
  </Step>
  
  <Step title="Active Listing">
    Approved listing is visible in public searches
    
    - Appears in GET `/api/listings` results
    - Users can view full details
    - Creator can update or mark as acquired
  </Step>
  
  <Step title="Acquired (Optional)">
    Creator marks listing as sold/rented
    
    - `acquiredStatus` set to `"sold"` or `"rented"`
    - Listing may still be visible but marked as unavailable
  </Step>
</Steps>

Optimized for common query patterns:


## Static Methods

### `findByApprovedStatus`

Helper method to query listings by approval status:

```javascript
// From models/listingModel.js:104-106
listingSchema.static("findByApprovedStatus", function (isApproved = true) {
  return this.find({ isApproved });
});
```

**Usage:**
```javascript
const approvedListings = await Listing.findByApprovedStatus(true);
const pendingListings = await Listing.findByApprovedStatus(false);
```

## Acquired Status Toggle Logic

The toggle endpoint uses smart logic based on listing type:

```javascript
// From controllers/listingController.js
if (status === "for-sale") {
  listing.acquiredStatus = listing.acquiredStatus === "sold" ? undefined : "sold";
} else if (status === "for-rent") {
  listing.acquiredStatus = listing.acquiredStatus === "rented" ? undefined : "rented";
}
```

## Image Upload Details

Images are uploaded via multipart form data:

- **Max images:** 10 per listing
- **Field name:** `images` (array)
- **Middleware:** `upload.array("images", 10)`
- **Storage:** Cloudinary with auto-optimization
- **Format:** URL and publicId stored for each image

## Query Filtering

The `/api/listings` endpoint supports extensive filtering:

```javascript
// From utils/buildQuery.js - supports:
- city (exact match, lowercase)
- minPrice, maxPrice (range)
- propertyType (exact match)
- minBedrooms, maxBedrooms (range)
- minBathrooms, maxBathrooms (range)
- minArea, maxArea (range in sqft)
- search (title search)
- sort (createdAt, price, areaSqft)
```

<Tip>Combine filters for precise searches: `?city=karachi&propertyType=apartment&minPrice=200000&maxPrice=500000`</Tip>
