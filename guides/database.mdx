Boundry Backend uses MongoDB with Mongoose for data persistence. This guide covers the database architecture, schemas, and optimization strategies.

## Connection Setup

Database connection is established on application startup:

```javascript
// From config/db.js:6-14
const connectDb = async () => {
  try {
    await mongoose.connect(process.env.MONGO_URI, { dbName: "boundry" });
    console.log("Successfully connected to MongoDB");
  } catch (e) {
    console.error("Couldn't connect to MongoDB ::", e.message);
    process.exit(1); // Exit if connection fails
  }
};
```

<Info>The database name is hardcoded as `"boundry"` - it's not derived from the URI</Info>

## Database Models

### User Model

Stores user accounts and authentication data.

**Schema Definition:**

```javascript
// From models/userModel.js:3-51
const userSchema = mongoose.Schema({
  username: { type: String, required: true, lowercase: true },
  email: { 
    type: String, 
    required: true, 
    unique: true, 
    lowercase: true, 
    trim: true,
    match: [/^[^\s@]+@[^\s@]+\.[^\s@]+$/, "Please enter a valid email"]
  },
  password: { 
    type: String, 
    required: true, 
    minlength: [8, "Password must be 8 characters long"] 
  },
  isAdmin: { type: Boolean, default: false },
  createdAt: { type: Date, default: Date.now },
  phone: { 
    type: String, 
    required: true, 
    unique: true,
    match: [/^\+?[1-9][0-9]{7,14}$/, "Please enter a valid Phone number"],
    trim: true 
  },
  avatar: {
    url: { type: String },
    publicId: { type: String, default: "" }
  }
});
```

**Indexes:**

```javascript
// From models/userModel.js:61-63
userSchema.index({ email: 1 }, { unique: true });
userSchema.index({ username: 1 }, { unique: true });
userSchema.index({ isAdmin: 1 });
```

**Virtual Fields:**

```javascript
// From models/userModel.js:53-58
userSchema.virtual("listings", {
  ref: "Listing",
  count: true,
  localField: "_id",
  foreignField: "owner",
});
```

<Tip>Virtuals are computed on-the-fly and not stored in the database</Tip>

### Listing Model

Stores property listings with location, pricing, and image data.

**Schema Definition:**

```javascript
// From models/listingModel.js:3-94
const listingSchema = mongoose.Schema({
  title: { type: String, required: true, trim: true },
  description: { type: String, required: true, trim: true },
  price: { type: Number, required: true, min: [1, "Price > 0"] },
  location: {
    city: { type: String, required: true, trim: true, lowercase: true },

    type: String,
    required: true,
    lowercase: true,

    type: String,

  bathrooms: { type: Number, required: true, min: [1, "Min 1 bathroom"] },
  areaSqft: { type: Number, required: true },
  images: [{
    url: { type: String, required: true },

  }],

    type: mongoose.Schema.Types.ObjectId, 
    ref: "User", 
    required: true
```

## Indexing Strategy

### Why Indexes Matter

Indexes dramatically improve query performance by avoiding full collection scans.

**Without index:**
```javascript
// O(n) - scans entire collection
db.listings.find({ city: "karachi" })
```

**With index:**
```javascript
// O(log n) - uses B-tree lookup
db.listings.find({ city: "karachi" })
```

## Query Patterns

### Filtering with buildQuery

The `buildQuery` utility constructs MongoDB queries from HTTP query parameters:

```javascript
// From utils/buildQuery.js:1-43
export default function buildQuery(queryParams) {
  const query = {};
  
  if (queryParams.city) {
    query["location.city"] = queryParams.city.toLowerCase();
  }
  
  if (queryParams.minPrice || queryParams.maxPrice) {
    query.price = {};
    if (queryParams.minPrice) query.price.$gte = Number(queryParams.minPrice);
    if (queryParams.maxPrice) query.price.$lte = Number(queryParams.maxPrice);
  }
  
  if (queryParams.propertyType) {
    query.propertyType = queryParams.propertyType.toLowerCase();
  }
  
  // Similar logic for bedrooms, bathrooms, area ranges
  
  return query;
}
```

**Example usage:**

```javascript
const query = buildQuery({
  city: "Karachi",
  minPrice: 200000,
  maxPrice: 500000,
  propertyType: "apartment"
});

const listings = await Listing.find(query);
```

Translates to:

```javascript
{
  "location.city": "karachi",
  "price": { $gte: 200000, $lte: 500000 },
  "propertyType": "apartment"
}
```

### Population (Joins)

MongoDB doesn't have SQL joins, but Mongoose provides `.populate()` for referencing documents:

```javascript
// From controllers/listingController.js
const listing = await Listing.findById(id).populate({
  path: "createdBy",
  select: "username email phone avatar",
});
```

Fetches the listing and includes the creator's details:

```json
{
  "_id": "...",
  "title": "Apartment",
  "createdBy": {
    "_id": "...",
    "username": "johndoe",
    "email": "john@example.com",
    "phone": "+123456789",
    "avatar": { "url": "..." }
  }
}
```

<Warning>Population adds extra queries - use sparingly in list endpoints</Warning>

## Pagination

Efficient pagination for large datasets:

```javascript
const page = parseInt(req.query.page) || 1;
const limit = Math.min(parseInt(req.query.limit) || 10, 100);
const skip = (page - 1) * limit;

const listings = await Listing.find(query)
  .skip(skip)
  .limit(limit)
  .sort({ createdAt: -1 });

const total = await Listing.countDocuments(query);

res.json({
  listings,
  currentPage: page,
  totalPages: Math.ceil(total / limit),
  total
});
```

<Tip>Always set a max limit (e.g., 100) to prevent performance issues from large page sizes</Tip>

## Backup and Restore

### Backup

```bash
# Backup entire database
mongodump --uri="mongodb://localhost:27017/boundry" --out=/backup/

# Backup specific collection
mongodump --uri="mongodb://localhost:27017/boundry" --collection=listings --out=/backup/
```

### Restore

```bash
# Restore entire database
mongorestore --uri="mongodb://localhost:27017/boundry" /backup/boundry/

# Restore specific collection
mongorestore --uri="mongodb://localhost:27017/boundry" --collection=listings /backup/boundry/listings.bson
```

## Connection Pooling

Mongoose maintains a connection pool automatically:

```javascript
// Default pool size is 5
mongoose.connect(uri, {
  maxPoolSize: 10, // Max 10 concurrent connections
  minPoolSize: 2,  // Keep 2 connections always ready
});
```

<Info>For most applications, the default pool size is sufficient</Info>
