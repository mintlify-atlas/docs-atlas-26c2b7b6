This guide will walk you through setting up Boundry Backend and making your first API request.

## Prerequisites

Before you begin, ensure you have:

- Node.js 16+ installed
- MongoDB instance (local or cloud)
- Cloudinary account for image uploads

## Installation

<Steps>
  <Step title="Clone the Repository">
    Clone the Boundry Backend repository and navigate to the project directory:
    
    ```bash
    git clone https://github.com/Raza-Aziz/Boundry-Backend.git
    cd Boundry-Backend
    ```
  </Step>
  
  <Step title="Install Dependencies">
    Install the required npm packages:
    
    <CodeGroup>
    ```bash npm
    npm install
    ```
    
    ```bash yarn
    yarn install
    ```
    
    ```bash pnpm
    pnpm install
    ```
    </CodeGroup>
    
    <Info>The project uses ES modules (`"type": "module"` in package.json)</Info>
  </Step>
  
  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory with the following variables:
    
    ```bash
    # Server Configuration
    PORT=5000
    NODE_ENV=development
    
    # Database
    MONGO_URI=mongodb://localhost:27017/boundry
    
    # JWT Authentication
    JWT_ACCESS_KEY=your_secret_access_key_here
    
    # Cloudinary Configuration
    CLOUDINARY_CLOUD_NAME=your_cloud_name
    CLOUDINARY_API_KEY=your_api_key
    CLOUDINARY_API_SECRET=your_api_secret
    ```
    
    <Warning>Never commit your `.env` file to version control. Keep your secrets secure.</Warning>
  </Step>
  
  <Step title="Start the Server">
    Run the development server with auto-reload:
    
    ```bash
    npm run backend
    ```
    
    You should see:
    ```
    Successfully connected to MongoDB
    App listening on port 5000
    ```
    
    The API is now running at `http://localhost:5000`
  </Step>
</Steps>

## Make Your First Request

Let's create a user account and authenticate.

### Register a New User

```bash
curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "johndoe",
    "email": "john@example.com",
    "password": "securepass123",
    "phone": "+1234567890"
  }'
```

<Accordion title="Response">
```json
{
  "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
  "username": "johndoe",
  "email": "john@example.com",
  "phone": "+1234567890",
  "isAdmin": false
}
```

The server automatically sets an HTTP-only JWT cookie named `jwt` in the response.
</Accordion>

### Login

```bash
curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -c cookies.txt \
  -d '{
    "email": "john@example.com",
    "password": "securepass123"
  }'
```

<Note>The `-c cookies.txt` flag saves the JWT cookie for subsequent requests</Note>

### Access Protected Routes

Use the saved cookie to access authenticated endpoints:

```bash
curl -X GET http://localhost:5000/api/auth/profile \
  -b cookies.txt
```

<Accordion title="Response">
```json
{
  "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
  "username": "johndoe",
  "email": "john@example.com",
  "phone": "+1234567890",
  "isAdmin": false,
  "avatar": {
    "url": "https://avatar.iran.liara.run/public/boy?username=johndoe",
    "publicId": ""
  },
  "createdAt": "2024-03-15T10:30:00.000Z"
}
```
</Accordion>

## Create Your First Listing

Now let's create a property listing with image upload:

```bash
curl -X POST http://localhost:5000/api/listings \
  -b cookies.txt \
  -F "title=Beautiful Downtown Apartment" \
  -F "description=Spacious 2-bedroom apartment in the heart of downtown" \
  -F "price=250000" \
  -F "location[city]=karachi" \
  -F "location[area]=Clifton" \
  -F "propertyType=apartment" \
  -F "status=for-sale" \
  -F "bedrooms=2" \
  -F "bathrooms=2" \
  -F "areaSqft=1200" \
  -F "images=@/path/to/image1.jpg" \
  -F "images=@/path/to/image2.jpg"
```

<Info>Images are uploaded to Cloudinary and optimized automatically</Info>

<Accordion title="Response">
```json
{
  "_id": "65f1a2b3c4d5e6f7a8b9c0d2",
  "title": "Beautiful Downtown Apartment",
  "description": "Spacious 2-bedroom apartment in the heart of downtown",
  "price": 250000,
  "location": {
    "city": "karachi",
    "area": "Clifton"
  },
  "propertyType": "apartment",
  "status": "for-sale",
  "bedrooms": 2,
  "bathrooms": 2,
  "areaSqft": 1200,
  "images": [
    {
      "url": "https://res.cloudinary.com/.../image1.jpg",
      "publicId": "boundry/abc123"
    },
    {
      "url": "https://res.cloudinary.com/.../image2.jpg",
      "publicId": "boundry/def456"
    }
  ],
  "createdBy": "65f1a2b3c4d5e6f7a8b9c0d1",
  "isApproved": false,
  "createdAt": "2024-03-15T10:35:00.000Z"
}
```

<Note>New listings require admin approval before appearing publicly</Note>
</Accordion>
